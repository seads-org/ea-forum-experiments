Analysis of EA Forum

Reports

1. Analysis of tags

We analysed various statistics over the tags and showed their pairwise correlations.

Results: 40% of tags have less than 10 posts, and most posts have no more than 3 tags. So, the signal is to weak to estimate proper correlations, but some tags can still be groupped.
Suggestion: (i) automated tagging would help with missing tags, or (ii) building hierarchies of tags would help users to search posts over broader categories.

2. Analysis of users

We analysed user posting and commenting patterns.

Results:
  • Most users (18k) didn't write a single post nor comment
  • Recent users are currently more active on the forum than users registered >1 year ago
  • People who regularly post on the forum usually have higher rating per post than those who are not active
  • Analysing user writing patterns we can see some clear groups of tags corresponding to the user clusters
Suggestion: (i) automated tagging would help with missing tags, or (ii) building hierarchies of tags would help users to search posts over broader categories.

3. Interactive exploration of semantic clusters

We groupped posts based on their semantic similarity and showed a two-level hierarchy of post topics.

Results:
  • The post clusters provide unbiased visualization of the forum content purely based on post texts.
  • The hierarchies can be used as a way to organize the forum in unsupervise manner.

4. Analysis of semantic clusters

We matched semantic clusters defined on the previous step with the forum tags and analysed user behavior based on the defined clusters.

Results: using semantic clusters provides clearer separation of the users and tags into categories

Suggestions

  • Auto-tagging, hierarchies of tags
  • Analysing preferences based on visits, reads and votes
  • User-network analysis: adding social component based on recommendations of interesting writers (twitter-style)
  • Extracting lists of experts per topic. That would allow to (i) reward active users, (ii) label them to highlight that they can be trusted, (iii) possibly more precise rating that would only use scores from experts.
  • Semantic search over the forum.
  • Automated extraction of org announcements, project proposal, etc.
  • Community health metrics
